Conceptos de Programación Orientada a Objetos

Herencia Polimorfismo Encapsulamiento Abstracción

1. Herencia

La herencia es un mecanismo que permite a una clase (llamada clase derivada o subclase) heredar características (atributos y métodos) de otra clase (llamada clase base o superclase). Esto promueve la reutilización del código y la creación de jerarquías de clases.


class Animal:
    def hacer_sonido(self):
        print("Sonido de animal")

class Perro(Animal):
    def hacer_sonido(self):
        print("Ladrido")

2. Polimorfismo

El polimorfismo permite a los objetos de diferentes clases ser tratados como objetos de una clase común. Se manifiesta en la capacidad de un método para tomar muchas formas. El polimorfismo se logra a través de la sobrecarga y la redefinición de métodos.


def hacer_sonido(animales):
    for animal in animales:
        animal.hacer_sonido()

perro = Perro()
animal = Animal()

hacer_sonido([perro, animal])

3. Encapsulamiento

El encapsulamiento es el concepto de ocultar los detalles internos de una clase y exponer solo lo necesario a través de una interfaz pública. Esto se logra utilizando modificadores de acceso y métodos de acceso (getters y setters) para proteger los datos y garantizar su integridad.


class CuentaBancaria:
    def __init__(self, saldo):
        self.__saldo = saldo
    
    def obtener_saldo(self):
        return self.__saldo
    
    def depositar(self, cantidad):
        if cantidad > 0:
            self.__saldo += cantidad

4. Abstracción

La abstracción se refiere a la capacidad de enfocarse en los aspectos esenciales de un objeto mientras se ocultan los detalles complejos. Permite a los programadores trabajar con una representación simplificada de un objeto.


from abc import ABC, abstractmethod

class Figura(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangulo(Figura):
    def __init__(self, ancho, alto):
        self.ancho = ancho
        self.alto = alto
    
    def area(self):
        return self.ancho * self.alto
